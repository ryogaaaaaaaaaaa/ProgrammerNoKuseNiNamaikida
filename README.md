# Alert Stack

Godot 4.6 で作った、リアルタイム・インシデント対応ローグライトのMVPです。  
プレイヤーは深夜シフトのインシデントコマンダーとなり、本番環境の依存グラフ上で広がる障害を、Alert Queue と Response Stack を裁きながら収束させます。

## 遊び方

- ゲームはリアルタイムで進みます。Spaceは倍速切り替えで、進行条件ではありません。
- まず右の `いま見るところ` を見ます。最重要インシデントと、次に押す候補が表示されます。
- 下の光っている `RUNBOOK COMMANDS` をクリックすると、対応が `RESPONSE STACK` に積まれます。
- 慣れてきたら中央の `SERVICE GRAPH` をクリックして、別サービスを手動で見ることもできます。
- 対応は即時発動ではなく、数秒かけてキュー順に完了します。
- `割り込み` をONにすると、次のRunbookを先頭に積めます。ただし疲労とStack Pressureが増えます。
- Error Budgetが0になると敗北。シフト終了まで耐えれば勝利です。

## Runbook

- ログを絞る: 原因確度を上げ、正しい対処を見極める
- レート制限: 負荷や攻撃を抑えるが、UXを少し削る
- 隔離: 伝播を止めるが、依存機能が一時停止する
- ロールバック: deploy/config起因に強いが、違う原因だと空振り
- 再起動: ヘルスを戻すが、一時的な不安定さを伴う
- 担当者を呼ぶ: 処理速度が上がるが、疲労が増える

## 操作

- 左クリック: サービス選択、Runbook投入、ボタン操作
- 右クリック / Esc: 選択解除
- Space: 倍速切り替え
- P: 一時停止
- F1: 遊び方
- M: ミュート切り替え

## 起動

```bash
godot --path .
```

Godot エディタから開く場合は、このフォルダをプロジェクトとしてインポートしてください。

## テスト

```bash
godot --headless --path . -s tests/smoke_test.gd
```
